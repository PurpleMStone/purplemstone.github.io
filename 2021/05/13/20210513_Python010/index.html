Pipfile

在用pipenv创建虚拟环境时，当前目录（不是虚拟环境所在目录）下会生成两个文件：

Pipfile

Pipfile.lock

Pipfile文件：

[[source]]
url = “https://pypi.org/simple“
verify_ssl = true
name = “pypi”

[packages]
requests = “*”

[dev-packages]
pylint = “*”

[requires]
python_version = “3.9”

Pipfile.lock是一个json文件，存了所有包的版本，所以换一台机器跑这个代码，依赖的还是这些版本的包，就不会出问题。
